## What is the MCP?
The <a href="https://modelcontextprotocol.io/introduction" target="_blank">Model Context Protocol (MCP)</a> is an open protocol that standardizes how applications provide context to AI models. 
MCP provides a standardized way to connect AI models to various data sources and tools, ensuring seamless integration and optimized performance. 
Think of MCP as a bridge between AI and your tools: it allows you to extend the functionality of Michelangelo.land by easily connecting it to different data sources via standardized interfaces.

## How to implement MCP?
MCP allows you to connect Michelangelo.land to external systems and data sources, making it easy to integrate with existing tools and infrastructures without the need for complex configurations. 
Simply define the MCP data in a JSON file, and the AI ​​chat will automatically take care of the integration based on your requests. <br />
You can manage and add MCPs directly from the ‘**MCP**’ **section** in the <a href="https://michelangelo.land" target="_blank">Michelangelo.land</a> development interface.

## Configuring MCP Servers
The MCP configuration uses a JSON format with the following structure:

```json
{
    "mcpServers": {
        "supabase": {
            "command": "прх",
            "args": [
                "-y", 
                "@modelcontextprotocol/server-postgres", 
                "<connection-string>"
            ]
        },
         "Framelink Figma MCP": {
            "command": "прх",
            "args": [
                "-y",
                "figna-developer-mcp",
                "--figma-api-key=YOUR-KEY",
                "--stdio"
            ]
        }
    }
}
```

## Limitations
MCP is a new protocol and is under development. There are some known limitations to be aware of:

    * some MCP servers, or users with many active MCP servers, may have many tools available for Michelangelo.land to use. Currently, Michelangelo.land will only send the first **40 tools to the Agent**.
    * Michelangelo.land communicates directly with MCP servers directly via `stdio` or `sse`.
